Doctor Booking API - Local Setup Guide

Welcome to the backend setup guide for the Doctor Booking application. Follow these steps carefully to get the backend server running on your local machine and connect it to your Flutter app.

Step 1: Install Python 3.12.2

To run this project, you need Python installed on your computer. We are using a specific version to ensure maximum compatibility.

  • Download the exact version here:
    Download Python 3.12.2 (Windows 64-bit)
⚠️ CRITICAL STEP: During the installation process, you MUST check the box that says "Add Python 3.12 to PATH" at the bottom of the installation window before clicking "Install Now".

Step 2: Download the Project

The backend project is hosted on GitHub. You need to download it to your local machine.

GitHub Repository: doctor_booking_app_backend

Open your terminal or command prompt and run the following command to clone the repository: 

git clone https://github.com/Mohammad-Alhajjar/doctor_booking_app_backend.git

Once the download is complete, navigate into the new doctor_booking_app_backend folder.

Important files inside the folder:
  • main.py: The actual Python backend code containing the API endpoints.
  • doctor_booking_app_collection.json: A Postman collection file you can import to easily test the APIs.

Step 3: Run the Backend Server

Now, let's start the server so your Flutter app can talk to it.

  1. Open the extracted project folder in Visual Studio Code (VS Code).
  2. Open a new Terminal inside VS Code (Terminal -> New Terminal).
  3. Install the required packages by running this command: 
    pip install fastapi uvicorn pydantic
  4. Start the server by running: 
    uvicorn main:app --reload

If successful, you will see a message saying: Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit).

Step 4: Connect Flutter to the API (The Localhost Rule)

Connecting your app to the backend depends on the platform you are using for testing:

📱 Android Emulator: You must change the base URL to http://10.0.2.2:8000. This is because the emulator sees 127.0.0.1 as its own internal address, while 10.0.2.2 points to your computer's localhost.

💻 iOS Simulator, Web, or Desktop: You can keep the default address: http://127.0.0.1:8000.

Step 5: Test with Postman

To verify everything is working, open the Postman app, click on Import, and select the doctor_booking_app_collection.json file you downloaded earlier. This will load all the pre-configured requests (Login, Book Doctor, etc.) so you can test them instantly!
