Doctor Booking API - دليل الإعداد المحلي

مرحباً بك في دليل إعداد الباك إيند لتطبيق Doctor Booking. اتبع هذه الخطوات بعناية لتشغيل سيرفر الباك إيند على جهازك المحلي وربطه بتطبيق فلاتر الخاص بك.

الخطوة 1: تثبيت بايثون 3.12.2

لتشغيل هذا المشروع، ستحتاج إلى تثبيت لغة بايثون على جهازك. نحن نستخدم إصداراً محدداً لضمان أقصى درجات التوافق.

  • قم بتحميل النسخة المطلوبة مباشرة من هنا:
    تحميل بايثون 3.12.2 (ويندوز 64-بت)
⚠️ خطوة هامة جداً: أثناء عملية التثبيت، يجب تفعيل خيار "Add Python 3.12 to PATH" في أسفل نافذة التثبيت قبل الضغط على "Install Now".

الخطوة 2: تحميل المشروع

مشروع الباك إيند مستضاف على GitHub. ستحتاج إلى تحميل نسخة منه على جهازك المحلي.

مستودع GitHub: doctor_booking_app_backend

افتح Terminal أو Command Prompt وقم بتشغيل الأمر التالي لنسخ المستودع (Clone): 

git clone https://github.com/Mohammad-Alhajjar/doctor_booking_app_backend.git

بمجرد اكتمال التحميل، انتقل إلى داخل المجلد الجديد doctor_booking_app_backend.

ملفات هامة داخل المجلد:
  • main.py: ملف كود بايثون الأساسي الذي يحتوي على روابط الـ API.
  • doctor_booking_app_collection.json: ملف مجموعة Postman يمكنك استيراده لاختبار الروابط بسهولة.

الخطوة 3: تشغيل سيرفر الباك إيند

الآن، لنقم بتشغيل السيرفر لكي يتمكن تطبيق فلاتر من الاتصال به.

  1. افتح مجلد المشروع المحمل باستخدام برنامج Visual Studio Code (VS Code).
  2. افتح Terminal جديد داخل VS Code من خلال (Terminal -> New Terminal).
  3. قم بتثبيت المكتبات المطلوبة عبر تشغيل هذا الأمر: 
    pip install fastapi uvicorn pydantic
  4. ابدأ تشغيل السيرفر عبر الأمر التالي: 
    uvicorn main:app --reload

إذا تمت العملية بنجاح، ستظهر لك رسالة مفادها: Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit).

الخطوة 4: ربط فلاتر بالـ API (قاعدة الـ Localhost)

يعتمد ربط تطبيقك بالباك إيند على المنصة التي تستخدمها للاختبار:

📱 محاكي أندرويد (Android Emulator): يجب تغيير رابط الأساس (Base URL) إلى http://10.0.2.2:8000. ذلك لأن المحاكي يرى العنوان 127.0.0.1 كعنوان داخلي خاص به، بينما العنوان 10.0.2.2 يشير إلى الـ localhost الخاص بجهاز الكمبيوتر.

💻 محاكي iOS، الويب، أو نسخة المكتب (Desktop): يمكنك الإبقاء على العنوان الافتراضي: http://127.0.0.1:8000.

الخطوة 5: الاختبار باستخدام Postman

للتأكد من أن كل شيء يعمل بشكل صحيح، افتح تطبيق Postman، واضغط على Import، ثم اختر ملف doctor_booking_app_collection.json الذي قمت بتحميله. سيقوم هذا بتحميل جميع الطلبات الجاهزة (تسجيل الدخول، حجز طبيب، إلخ) لتجربتها فوراً!
